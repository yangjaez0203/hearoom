# 기능별 커밋

현재 변경사항을 분석하고 기능별로 분리하여 커밋합니다.

## 인자

```
$ARGUMENTS
```

- 인자가 있으면: 해당 기능에 대한 변경사항만 커밋
- 인자가 없으면: 변경사항을 분석하여 기능을 추론하거나 사용자에게 질문

## 커밋 메시지 형식

```
[{노션아이디}] {타입}: {설명}
```

예시: `[ABC-123] feat: 로그인 기능 추가`

## 커밋 타입

- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 변경
- style: 코드 포맷팅 (기능 변경 없음)
- refactor: 코드 리팩토링
- test: 테스트 추가/수정
- chore: 빌드, 설정 등 기타 변경

## 수행 단계

1. `git status`와 `git diff`를 실행하여 현재 변경사항 확인
2. **노션 아이디를 모르면 반드시 사용자에게 물어볼 것**
3. 인자가 있으면 해당 기능 관련 파일만 선택, 없으면 변경사항에서 기능 추론 또는 사용자에게 질문
4. 커밋 메시지 제안 및 사용자 확인
5. 승인 후 커밋 생성

## 중요

- 노션 아이디가 없으면 커밋하지 말고 사용자에게 먼저 물어보세요
- 커밋할 기능이 불분명하면 사용자에게 물어보세요
- 각 커밋 전에 사용자 확인을 받으세요
